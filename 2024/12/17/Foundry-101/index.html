有四种测试

  • Unit tests: Focus on isolating an d testing individual smart contract functions or functionalities.
  • Integration tests: Verify how a smart contract interacts with other contracts or external systems.
  • Forking tests: Forking refers to creating a copy of a blockchain state at a specific point in time. This copy, called a fork, is then used to run tests in a simulated environment.
  • Staging tests: Execute tests against a deployed smart contract on a staging environment before mainnet deployment.

Very important note: the word private has multiple meanings in the Merriam Webster dictionary. Always remember that the keyword private attached to a variable/function means that the variable/function has restricted access from other parties apart from the main contract.

THIS DOESN’T MEAN THE VARIABLE IS A SECRET!!!

Everything on the blockchain is public. Any smart contract’s storage can be accessed in one of the 3 ways we showed in this lesson. Do not share sensitive information or store passwords inside smart contracts, we can all read them.

vm.startBroadcast()

  1. function startBroadcast() external;
  2. function startBroadcast(address who) external;
  3. function startBroadcast(uint256 privateKey) external;

这些函数的作用是开始一个“广播”模式，在这个模式下，所有后续的函数调用将会生成可以签名并发送到链上的交易。

  • startBroadcast()：通过调用该函数，所有后续的合约调用会以当前调用者的地址作为发送者，并生成交易（在当前调用深度中，且不包括对某些特殊代码（cheat code）的调用）。
  • startBroadcast(address who)：通过指定一个地址 who，所有后续的合约调用会以这个地址作为发送者，并生成交易。
  • startBroadcast(uint256 privateKey)：通过指定一个私钥 privateKey，所有后续的合约调用会使用与这个私钥对应的地址作为发送者，并生成交易。

**vm.etch() **是 Foundry 的虚拟机（vm）提供的一个特殊方法，用来将合约代码直接写入到指定地址。它的作用是直接在某个地址上“部署”一段合约代码，或者说替换某个地址上的代码。

  • target: 目标地址，即你希望在哪个地址上部署或替换合约代码。
  • code: 你希望部署到该地址的合约代码，以字节码的形式。
