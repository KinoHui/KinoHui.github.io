## Windows 环境

```bash
go run .\cmd\kube-apiserver\apiserver.go \
  --etcd-servers=http://127.0.0.1:2379 \
  --client-ca-file=D:\k8s-sorce-code-ref\cert\ca.crt \
  --tls-cert-file=D:\k8s-sorce-code-ref\cert\server.crt \
  --tls-private-key-file=D:\k8s-sorce-code-ref\cert\server.key \
  --service-account-issuer=api \
  --service-account-key-file=D:\k8s-sorce-code-ref\cert\ca.crt \
  --service-account-signing-key-file=D:\k8s-sorce-code-ref\cert\ca.key \
  --kubelet-client-certificate=D:\k8s-sorce-code-ref\cert\server.crt \
  --kubelet-client-key=D:\k8s-sorce-code-ref\cert\server.key
```

## WSL2 环境

先关闭 WSL2 本地 k3s 集群：

```bash
sudo systemctl disable k3s
```

然后启动：

```bash
go run ./cmd/kube-apiserver/apiserver.go \
  --etcd-servers=http://127.0.0.1:2379 \
  --client-ca-file=/mnt/d/k8s-sorce-code-ref/cert/ca.crt \
  --tls-cert-file=/mnt/d/k8s-sorce-code-ref/cert/server.crt \
  --tls-private-key-file=/mnt/d/k8s-sorce-code-ref/cert/server.key \
  --service-account-issuer=api \
  --service-account-key-file=/mnt/d/k8s-sorce-code-ref/cert/ca.crt \
  --service-account-signing-key-file=/mnt/d/k8s-sorce-code-ref/cert/ca.key \
  --kubelet-client-certificate=/mnt/d/k8s-sorce-code-ref/cert/server.crt \
  --kubelet-client-key=/mnt/d/k8s-sorce-code-ref/cert/server.key
```
