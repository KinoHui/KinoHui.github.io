大家好，本篇教程将**从零开始**，手把手带你在 Ubuntu 系统中部署一套完整的 Kubernetes 1.31 集群，包含 **Control-Plane（控制平面）** 和 **Worker（工作节点）** 全流程，使用 Containerd 作为容器运行时、Calico 作为网络插件，步骤清晰、可直接复刻使用！

---

## 一、环境前置说明

1. 系统：Ubuntu（Debian 系通用）
2. K8s 版本：v1.31.14
3. 容器运行时：containerd 1.7.23
4. 网络插件：Calico v3.28.2
5. 节点规划：1 个 Control-Plane 节点 + 任意数量 Worker 节点
6. 前置要求：所有节点**关闭防火墙、SELinux、Swap 分区**，节点之间网络互通

---

## 二、通用前置配置（所有节点必须执行！）

Control-Plane 和 Worker 节点的**基础环境、依赖、容器运行时配置完全一致**，先在所有节点执行以下步骤：

### 1. 开启 IP 转发（K8s 网络必需）

K8s 集群依赖 IP 转发实现容器网络通信，必须永久开启：

```bash
# 临时开启 IP 转发
sysctl net.ipv4.ip_forward

# 永久写入配置文件
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.ipv4.ip_forward = 1
EOF

# 加载系统配置
sudo sysctl --system

# 验证是否生效（输出值应为1）
sysctl net.ipv4.ip_forward
```

### 2. 创建安装目录，下载 Containerd 相关组件

```bash
# 创建安装目录
mkdir k8s-cluster-installs
cd k8s-cluster-installs/

# 1. 下载 containerd
wget https://github.com/containerd/containerd/releases/download/v1.7.23/containerd-1.7.23-linux-amd64.tar.gz
sudo tar Cxzvf /usr/local containerd-1.7.23-linux-amd64.tar.gz

# 2. 配置 containerd systemd 服务
sudo mkdir -p /usr/local/lib/systemd/system/
wget https://raw.githubusercontent.com/containerd/containerd/main/containerd.service
sudo mv containerd.service /usr/local/lib/systemd/system/

# 3. 下载 runc（容器运行时依赖）
wget https://github.com/opencontainers/runc/releases/download/v1.1.13/runc.amd64
sudo install -m 755 runc.amd64 /usr/local/sbin/runc

# 4. 下载 CNI 插件（容器网络接口）
wget https://github.com/containernetworking/plugins/releases/download/v1.5.1/cni-plugins-linux-amd64-v1.5.1.tgz
sudo mkdir -p /opt/cni/bin
sudo tar Cxzvf /opt/cni/bin cni-plugins-linux-amd64-v1.5.1.tgz
```

### 3. 配置 Containerd 并启动

```bash
# 创建 containerd 配置目录
sudo mkdir -p /etc/containerd

# 生成默认配置文件
containerd config default | sudo tee /etc/containerd/config.toml

# 关键配置：启用 SystemdCgroup（K8s 强制要求）
sudo sed -i 's/SystemdCgroup \= false/SystemdCgroup \= true/g' /etc/containerd/config.toml

# 重载 systemd 并启动 containerd，设置开机自启
sudo systemctl daemon-reload
sudo systemctl enable --now containerd

# 验证 containerd 状态（显示 active (running) 即为成功）
systemctl status containerd
```

### 4. 重启 Containerd（加载自定义配置）

如果你有自定义的 `config.toml`，执行以下命令覆盖配置并重启：

```bash
# 替换自定义配置（无自定义可跳过）
sudo rm /etc/containerd/config.toml
sudo cp config.toml /etc/containerd/config.toml

# 重启 containerd 使配置生效
sudo systemctl restart containerd
```

### 5. 安装 K8s 核心组件（kubelet/kubeadm/kubectl）

```bash
# 更新 apt 源
sudo apt-get update

# 安装依赖工具
sudo apt-get install -y apt-transport-https ca-certificates curl gpg

# 添加 K8s 官方 GPG 密钥
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.31/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

# 添加 K8s apt 软件源
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.31/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list

# 再次更新源，安装 K8s 组件
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl

# 锁定版本，防止自动更新
sudo apt-mark hold kubelet kubeadm kubectl

# 安装网络依赖工具
sudo apt-get install -y conntrack
```

✅ **至此，所有节点的通用配置完成！** 接下来分节点执行专属操作。

---

## 三、Control-Plane 节点专属配置（仅主节点执行）

### 1. 初始化 K8s 控制平面

这是集群核心步骤，**替换命令中的 IP 为你的 Control-Plane 节点实际 IP**：

```bash
sudo kubeadm init \
--apiserver-advertise-address=192.168.59.134 \
--image-repository registry.aliyuncs.com/google_containers \
--kubernetes-version v1.31.14 \
--service-cidr=10.96.0.0/12 \
--pod-network-cidr=10.244.0.0/16
```

- `apiserver-advertise-address`：主节点 IP
- `image-repository`：阿里云镜像源（解决国内拉取镜像失败）
- `pod-network-cidr`：Pod 网段，必须和后续 Calico 配置一致

### 2. 配置 kubectl 权限（普通用户使用 K8s）

初始化成功后，执行以下命令，让当前用户可以使用 `kubectl` 命令：

```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

### 3. 取消主节点污点（可选，允许主节点运行 Pod）

默认 K8s 主节点不运行业务 Pod，执行命令取消限制：

```bash
kubectl taint nodes --all node-role.kubernetes.io/control-plane-
```

### 4. 部署 Calico 网络插件

K8s 必须安装网络插件才能实现节点间通信，这里使用 Calico：

```bash
# 部署 Calico
kubectl apply -f https://raw.githubusercontent.com/projectcalico/calico/v3.28.2/manifests/calico.yaml

# 配置 Calico Pod 网段（和初始化时的 pod-network-cidr 保持一致）
kubectl patch daemonset calico-node -n kube-system -p '{"spec":{"template":{"spec":{"containers":[{"name":"calico-node","env":[{"name":"CALICO_IPV4POOL_CIDR","value":"10.244.0.0/16"}]}]}}}}'
```

### 5. 验证集群状态

```bash
# 查看 kube-system 命名空间下的 Pod，等待所有 Pod 运行成功
kubectl get pods -n kube-system

# 测试部署 Nginx 容器
kubectl run nginx --image=nginx:alpine
```

## 四、Worker 节点专属配置（仅工作节点执行）

### 1. 加入 K8s 集群

在 **Control-Plane 节点执行** **`kubeadm init`** **后**，会生成 `kubeadm join` 命令，直接复制到所有 Worker 节点执行：

```bash
sudo kubeadm join 192.168.59.134:6443 \
--token 0lcb2p.818vc1u4mj220yjq \
--discovery-token-ca-cert-hash sha256:fe61f6f1377f5e39f5e685c2b038eac956e3d40c631872444c0de4f58c5c30dc
```

执行成功后，回到 Control-Plane 节点，执行 `kubectl get nodes` 即可看到 Worker 节点已加入集群！

---

## 五、集群最终验证

在 Control-Plane 节点执行以下命令，确认集群正常：

```bash
# 查看所有节点状态（STATUS 为 Ready 即为成功）
kubectl get nodes

# 查看所有 Pod 状态（所有 Pod 为 Running）
kubectl get pods -A

# 查看测试 Nginx Pod
kubectl get pods
```

---

## 六、关键注意事项

1. **Token 过期**：如果 `kubeadm join` 的 token 过期，在主节点执行 `kubeadm token create --print-join-command` 重新生成。
2. **网络问题**：所有节点必须关闭 Swap、防火墙，保证 IP 互通。
3. **版本一致**：K8s 组件、Containerd、Calico 版本必须严格匹配教程。
4. **镜像加速**：教程已使用阿里云镜像源，国内环境可直接拉取镜像。

## 总结

本篇教程完整覆盖了 **K8s 1.31 集群部署的全流程**：

1. 所有节点统一配置 IP 转发、安装 Containerd 运行时；
2. 安装 K8s 核心组件，锁定版本；
3. 主节点初始化集群、配置权限、部署 Calico 网络；
4. 工作节点一键加入集群；
5. 最终验证集群可用性。

按照步骤执行，零基础也能快速搭建一套生产级可用的 K8s 集群！

### 核心要点回顾

1. **通用步骤**：所有节点必须完成 IP 转发、Containerd 配置、K8s 组件安装；
2. **主节点核心**：`kubeadm init` 初始化集群 + 部署 Calico 网络；
3. **工作节点核心**：执行 `kubeadm join` 加入集群，无额外配置；
4. **验证标准**：`kubectl get nodes` 所有节点为 `Ready`，集群即部署成功。
