这是我的 Hugo + PaperMod 博客项目。草稿放在 ./staging/ 目录，请帮我发布新文章。

请按下面流程操作：

1. 先检查 ./staging/ 下的新草稿，判断它属于哪个主题目录和系列。
2. 把草稿整理成正式文章，放到 content/posts/<topic>/ 下。
3. 文件名使用两位序号开头，例如 04-xxx.md。
4. 补全 front matter：
   - title
   - date，使用当前日期时间和 +08:00
   - draft: false
   - tags
   - categories
   - series
   - summary
   - weight
5. 如果是新系列：
   - 在 content/series/<series-slug>/_index.md 创建系列页
   - 写清 title 和 description
6. 如果是已有系列：
   - 检查同系列文章的 weight，给新文章设置正确顺序
   - 确保标题序号和 weight 对得上
7. 清理 Markdown：
   - 把 <pid>、<pod>、<container-id> 这类占位符放进反引号
   - 避免 raw HTML
   - 代码块使用正确语言标记
   - 表格使用标准 Markdown
8. 检查文章里的相对链接、relref、代码块和标题层级。
9. 运行 hugo --gc --minify。
10. 如果构建有 warning 或 error，修复后重新构建。
11. 最后总结：
   - 新增/修改了哪些文件
   - 文章发布路径
   - 所属 series、weight、tags、categories
   - 构建结果

不要修改 themes/PaperMod 目录。优先使用 layouts、assets、content 下的站点级 override。
